# คู่มือ Admin KaonA-Agri

เอกสารนี้สำหรับผู้ดูแลระบบ (Admin) เพื่อใช้งานระบบหลักในแต่ละวัน ตั้งแต่จัดการสมาชิกจนถึงสรุปปิดรับประจำวัน

---

## 1) การจัดการสมาชิก

### 1.1 Member approval
1. เข้าเมนู **Admin > Members > Pending**
2. เปิดข้อมูลสมาชิกที่รออนุมัติ
3. ตรวจความครบถ้วนของข้อมูลสำคัญ เช่น ชื่อ เบอร์โทร LINE และข้อมูลพื้นฐานเกษตรกร
4. เลือกผลการพิจารณา

### 1.2 Approve / Reject flow
- **Approve**
  1. กดอนุมัติ
  2. ตรวจว่าสถานะเปลี่ยนเป็นอนุมัติแล้ว
  3. ตรวจว่าสมาชิกหลุดจากรายการ pending
- **Reject**
  1. กดปฏิเสธ
  2. ระบุเหตุผลให้ชัดเจน
  3. ตรวจว่าสถานะเป็น rejected และระบบบันทึกเหตุผลได้

### 1.3 Import CSV สมาชิก
1. ไปที่ **Admin > Members > Import CSV**
2. อัปโหลดไฟล์ `.csv` ตาม template ที่กำหนด
3. ตรวจหน้าพรีวิวก่อนยืนยัน (valid / invalid rows)
4. กดยืนยันนำเข้า
5. ตรวจผลหลังนำเข้าและสุ่มเปิดข้อมูล 2-3 รายการเพื่อ QA

---

## 2) ตั้งราคารับซื้อ

### 2.1 ตั้งราคาฐาน (`market_prices`)
1. ไปที่เมนู **Admin > Pricing / Market Price**
2. เลือกวันที่มีผลบังคับใช้
3. กรอกราคาฐานต่อหน่วย
4. บันทึกและตรวจว่าแสดงในตารางราคาล่าสุด

### 2.2 ตั้งส่วนลดตามความชื้น (`moisture_deductions`)
1. ไปที่เมนู **Admin > Pricing > Moisture Deductions**
2. กำหนดช่วงความชื้นและอัตราหัก
3. ตรวจไม่ให้ช่วงทับกัน
4. บันทึกและทดสอบตัวอย่างคำนวณ

### 2.3 สร้างประกาศแคมเปญ (`campaign_announcements`)
1. ไปที่ **Admin > Campaigns / Announcements**
2. กรอกชื่อแคมเปญ เงื่อนไข ระยะเวลา
3. เลือกกลุ่มเป้าหมาย
4. Publish และตรวจสถานะ active

---

## 3) จัดการโควต้าและคิวรับซื้อ

### 3.1 ตั้ง template รายวัน (`intake_quota_templates`)
1. ไปที่ **Admin > Harvest > Quota Templates**
2. กำหนด capacity แยกวัน/จุดรับซื้อ
3. บันทึก template

### 3.2 สร้าง pickup slot จาก template
1. เลือก template ที่ต้องการใช้
2. ระบุช่วงวันที่สร้าง slot
3. ตรวจจำนวน slot และ capacity ต่อ slot
4. ยืนยันสร้าง

### 3.3 ดู realtime utilization
1. เปิดหน้า **Harvest Queue / Intake Board**
2. ตรวจแถบ utilization ของแต่ละจุดรับซื้อ
3. เฝ้าระวังวันที่เกิน threshold และจัดการกระจายคิว

---

## 4) Factory API Keys

### 4.1 สร้าง API key สำหรับเครื่องชั่ง
1. ไปที่ **Admin > Harvest > API Keys**
2. กดสร้าง key ใหม่
3. ตั้งชื่อ/คำอธิบายของ key ตามเครื่องหรือหน้างาน
4. คัดลอก key และส่งให้ผู้รับผิดชอบเครื่องชั่งผ่านช่องทางปลอดภัย

### 4.2 ปิดใช้งาน key
1. ค้นหา key ที่ต้องการปิด
2. กด Disable/Deactivate
3. ตรวจสถานะว่าไม่ active แล้ว
4. บันทึกเหตุผลการปิดใช้งานเพื่อ audit

---

## 5) Onboarding reset / check

### 5.1 ตรวจ onboarding checklist
1. เปิดโปรไฟล์สมาชิก
2. ตรวจสถานะแต่ละขั้นของ onboarding
3. ระบุขั้นที่ยังไม่ครบและแจ้งสมาชิก

### 5.2 รีเซ็ต onboarding
ใช้เฉพาะกรณีจำเป็น เช่น ข้อมูลผิดชุดใหญ่หรือสถานะ checklist ค้างผิด
1. ยืนยันตัวตนสมาชิก
2. แจ้งผลกระทบก่อนรีเซ็ต
3. สั่งรีเซ็ตจากเครื่องมือ admin
4. ให้สมาชิกทำ onboarding ใหม่
5. ตรวจซ้ำหลังสมาชิกทำครบ

---

## 6) ปิดรับวัน (Reconciliation)

### 6.1 ขั้นตอนปิดรับประจำวัน
1. ตรวจว่ารายการรับซื้อค้างถูกจัดการครบ
2. ตรวจยอดรวมรับซื้อประจำวัน
3. ตรวจความถูกต้องของ ticket สำคัญ
4. กดปิดรับวัน

### 6.2 Export CSV
1. ไปที่หน้า **Reconciliation / Reports**
2. เลือกวันที่ต้องการ export
3. ดาวน์โหลดไฟล์ CSV
4. ตรวจจำนวนแถวและยอดรวมเทียบกับหน้ารายงาน

---

## 7) รายงานที่ต้องติดตาม

### 7.1 รายงานสมาชิก (members)
- ใช้ติดตามจำนวนสมาชิกทั้งหมด สมาชิกที่อนุมัติแล้ว และคิว pending

### 7.2 รายงานการจองขาย (booking)
- ใช้ติดตามปริมาณการจอง สถานะคิว และช่วง peak

### 7.3 รายงานคาด vs จริง (expected vs actual)
- ใช้เทียบข้อมูลคาดการณ์กับข้อมูลรับจริง
- ใช้ตรวจความแม่นยำแผนและปรับการวางคิว

### 7.4 รายงานตามรถ / ตามพื้นที่ (vehicle / area)
- ใช้ติดตามประสิทธิภาพตามคันรถ
- ใช้ติดตามภาพรวมตามพื้นที่รับซื้อ

---

## 8) Troubleshooting

### อนุมัติสมาชิกแล้วแต่เข้าใช้งานไม่ได้
- ตรวจ role ว่าถูกต้อง
- ตรวจสถานะ approval ว่าบันทึกสำเร็จ
- ให้สมาชิกปิด-เปิด LIFF ใหม่และทดสอบอีกครั้ง

### Approve/Reject ไม่ผ่าน
- ตรวจสิทธิ์ผู้ใช้ว่าเป็น admin
- ตรวจ API/DB connectivity
- ตรวจ server logs เพื่อหา error code

### CSV import ไม่ผ่าน
- ตรวจหัวคอลัมน์และรูปแบบไฟล์ UTF-8
- ตรวจข้อมูลซ้ำ/ข้อมูลบังคับขาด
- ทดลองนำเข้าทีละชุดเล็กเพื่อตัดปัญหา

### Utilization หรือรายงานดูผิดปกติ
- ตรวจช่วงวันที่ filter
- ตรวจว่าปิดรับวันแล้วหรือยัง
- ตรวจข้อมูลต้นทางจาก intake/booking

---

## 9) Permissions / Roles (สรุป)

- **Admin**: จัดการสมาชิก ราคา โควต้า API key ปิดรับวัน รายงาน
- **Staff**: บันทึกรับซื้อ ตรวจคิวหน้างาน
- **Farmer**: สมัครสมาชิก จองขาย และติดตามข้อมูลของตน

> หากไม่เห็นเมนูที่ควรมี ให้ตรวจ role mapping และ session ล่าสุดก่อนเสมอ
