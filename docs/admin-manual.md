# คู่มือ Admin KaonA-Agri

เอกสารนี้สำหรับผู้ดูแลระบบ (Admin) เพื่อใช้งานงานหลักที่จำเป็นต่อการปฏิบัติการประจำวันในระบบ KaonA-Agri

---

## 1) บทบาทและสิทธิ์ (Permissions / Roles)

### บทบาทหลัก
- **admin**: จัดการสมาชิก อนุมัติ/ปฏิเสธ จัดการข้อมูลระบบ และตรวจสอบรายงาน
- **staff**: ปฏิบัติงานหน้างาน เช่น บันทึกรับซื้อและข้อมูลชั่ง
- **farmer**: ใช้งานฝั่งสมาชิก เช่น สมัครสมาชิก จองวันเกี่ยว ดูสถานะ/รายงานของตน

### สิทธิ์ที่เกี่ยวข้องกับงาน Admin
- อนุมัติหรือปฏิเสธสมาชิกใหม่
- ตรวจสอบและนำเข้าข้อมูลสมาชิกจาก CSV
- ตรวจสอบสถานะ onboarding และรีเซ็ตเมื่อจำเป็น
- เข้าถึงหน้า admin reports และหน้า system map

> หากผู้ใช้มองไม่เห็นเมนู admin ให้ตรวจสอบ role ในฐานข้อมูลก่อนเป็นอันดับแรก

---

## 2) การจัดการสมาชิก (Member Approval)

### 2.1 ตรวจสอบคำขอสมาชิกใหม่
1. เข้าเมนู **Admin > Members (Pending)**
2. เปิดรายการที่สถานะรออนุมัติ
3. ตรวจสอบข้อมูลขั้นต่ำที่ต้องครบ:
   - ชื่อ-นามสกุล
   - เบอร์โทร
   - ข้อมูลติดต่อ LINE
   - ข้อมูลพื้นฐานการใช้งาน

### 2.2 Approve / Reject Flow
1. เปิดรายละเอียดสมาชิกรายที่ต้องการพิจารณา
2. เลือกคำสั่ง:
   - **Approve**: เปลี่ยนสถานะเป็นอนุมัติ และให้สมาชิกเริ่ม onboarding ต่อได้
   - **Reject**: ปฏิเสธคำขอ โดยระบุเหตุผลให้ชัดเจน
3. ตรวจสอบผลลัพธ์หลังบันทึก:
   - สถานะสมาชิกอัปเดตถูกต้อง
   - รายการหายจากคิว pending ตาม expected
   - ระบบแจ้งเตือน/ข้อความผลการพิจารณาไปยังสมาชิกได้

### แนวปฏิบัติแนะนำ
- ใส่เหตุผลทุกครั้งเมื่อ Reject เพื่อให้สมาชิกแก้ไขได้ถูกจุด
- หากข้อมูลไม่พอพิจารณา ให้ติดต่อสมาชิกก่อน ไม่ควร approve แบบข้อมูลไม่ครบ

---

## 3) นำเข้าข้อมูลสมาชิกด้วย CSV (CSV Import)

### 3.1 เตรียมไฟล์
- ใช้ไฟล์ `.csv` ตาม template ที่ทีมกำหนด
- ใช้หัวคอลัมน์ให้ตรง 100%
- ตรวจสอบ encoding เป็น UTF-8

### 3.2 ขั้นตอนนำเข้า
1. เข้าเมนู **Admin > Members > Import CSV**
2. อัปโหลดไฟล์ CSV
3. ตรวจสอบหน้าพรีวิวก่อนยืนยัน (review)
4. ยืนยันนำเข้า (confirm import)

### 3.3 สิ่งที่ต้องตรวจในหน้า Preview
- จำนวนแถวที่ valid / invalid
- เบอร์โทรซ้ำ
- ข้อมูลบังคับที่หายไป (missing field)
- รูปแบบข้อมูลผิด (เช่น เบอร์โทรไม่ถูก format)

### 3.4 หลัง import
- ตรวจสอบจำนวนสมาชิกที่เพิ่มสำเร็จ
- สุ่มเปิดรายการใหม่ 2-3 รายการเพื่อตรวจความถูกต้อง
- บันทึกเวลาและผู้ดำเนินการทุกครั้ง

---

## 4) Onboarding: ตรวจสอบและรีเซ็ต (Reset / Check)

### 4.1 ตรวจสอบความคืบหน้า onboarding
1. เข้าโปรไฟล์สมาชิก
2. ดู onboarding checklist ว่าครบทุกขั้นหรือไม่
3. ขั้นที่มักพบว่ายังค้าง:
   - ข้อมูลโปรไฟล์
   - ข้อมูลแปลง
   - ข้อมูลรอบปลูก
   - ยืนยันความพร้อมใช้งาน

### 4.2 รีเซ็ต onboarding เมื่อจำเป็น
ใช้เมื่อเกิดกรณีต่อไปนี้เท่านั้น:
- สมาชิกกรอกข้อมูลผิดชุดใหญ่และต้องเริ่มใหม่
- สถานะ checklist ค้างผิดจากความจริง
- migration หรือ data patch กระทบสถานะเดิม

ขั้นตอน:
1. ยืนยันตัวตนสมาชิกให้ถูกคน
2. แจ้งผลกระทบก่อนรีเซ็ต (ข้อมูลบางส่วนอาจต้องกรอกใหม่)
3. สั่งรีเซ็ต onboarding จากเครื่องมือ admin
4. ให้สมาชิกเข้าทำ checklist ใหม่
5. ตรวจซ้ำสถานะหลังสมาชิกทำครบ

---

## 5) Troubleshooting

### ปัญหา: อนุมัติแล้วแต่สมาชิกยังใช้งานไม่ได้
- ตรวจ role ว่าถูกตั้งเป็น farmer/member ถูกต้องหรือไม่
- ตรวจสถานะ approval ว่าเขียนสำเร็จจริง
- ให้สมาชิกปิด-เปิด LIFF ใหม่ และลองเข้าอีกครั้ง

### ปัญหา: กด Approve/Reject ไม่สำเร็จ
- ตรวจสิทธิ์ของผู้ใช้งานว่าเป็น admin จริง
- ตรวจการเชื่อมต่อฐานข้อมูล/API
- ดู log ฝั่งเซิร์ฟเวอร์เพื่อหา error code

### ปัญหา: CSV import ล้มเหลว
- ตรวจหัวคอลัมน์ผิดชื่อหรือไม่
- ตรวจ delimiter (comma) และ encoding (UTF-8)
- ตรวจข้อมูลบังคับหาย/ซ้ำ/รูปแบบผิด
- ลอง import ด้วยไฟล์ขนาดเล็กเพื่อแยกปัญหา

### ปัญหา: Onboarding ไม่ขยับทั้งที่สมาชิกทำครบ
- ตรวจข้อมูลอ้างอิงที่ checklist ใช้คำนวณ
- ตรวจ cache/session ฝั่ง LIFF
- ตรวจว่าไม่มีข้อมูลบางตารางถูกลบหรือไม่ครบ

---

## 6) Checklist งาน Admin ประจำวัน

- เปิดดูสมาชิกค้างอนุมัติ และเคลียร์คิว
- ตรวจรายการที่ reject ว่ามีเหตุผลครบทุกเคส
- ตรวจผล import CSV ล่าสุด (ถ้ามี)
- สุ่มตรวจ onboarding ที่ค้างเกิน SLA
- ตรวจ error logs สำคัญก่อนจบวัน

---

## 7) แนวทางความปลอดภัยข้อมูล

- ใช้บัญชี admin เฉพาะผู้ได้รับมอบหมาย
- ห้ามแชร์บัญชีร่วมกัน
- ห้ามนำไฟล์ CSV สมาชิกออกนอกระบบโดยไม่จำเป็น
- บันทึก audit trail ทุกงานสำคัญ (approve/reject/import/reset)
